Debugging - Date: 01/30/15

Phases

  1. Where
  2. What
  3. How to fix

Strategies

  • Guess & Check (worst)
  • Analysis (not as bad)
  • Logarithm | Subdivide (if no line number)
  • Sandboxing

Tips

  • Consistently reproduce the bug
  • Identify the issue before you try to debug it
  • Test your assumptions
  • Fix first bug first
  • Understand the error message
  • Use $(...).length to test results
  • Mentally walk through with sample value

Tools

  • console.log statements
  • alert - works well with event.preventDefault()
  • preserve log - in the developer's console window
  • sources window - in the developer tool

Preventative

  • Write pure function
  • Encapsulate your logic
  • Implement unit testing framework
